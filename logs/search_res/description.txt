# 搜索结果数据目录

**路径**: `logs/search_res/`

## 目录概述
本目录存储银行信贷业务流程中大模型应用场景的搜索结果数据，包含从公开渠道收集的银行大模型项目信息汇总。数据主要用于分析银行信贷业务数字化转型趋势、大模型应用场景分布以及市场竞争格局。

## 文件结构

### 1. `search_items.csv` - 搜索结果明细表
- **文件格式**: CSV (逗号分隔值)
- **数据量**: 131行记录
- **更新频率**: 根据搜索任务执行情况更新
- **字段结构**:
  - `BankName`: 银行名称 - 字符串类型，记录银行全称
  - `Scene`: 应用场景 - 字符串类型，描述大模型在信贷业务中的具体应用场景
  - `Description`: 项目描述 - 字符串类型，详细的项目背景、内容和目标
  - `Type`: 采购类型 - 字符串类型，如"招标采购"、"公开招标"、"单一来源采购"等
  - `TimeCycle`: 时间周期 - 字符串类型，记录项目的时间信息（如"2025年7月"）
  - `Vendor`: 供应商 - 字符串类型，中标或合作的科技公司名称

- **数据特点**:
  - 覆盖国有银行、股份制银行、城商行、农商行等各类银行机构
  - 聚焦信贷业务流程中的大模型应用场景
  - 包含项目预算、中标金额等关键商业信息
  - 时间范围主要集中在2024-2025年

### 2. `search_items.json` - 搜索策略与过程记录
- **文件格式**: JSON
- **数据结构**: 包含多个智能体的搜索任务记录
- **主要内容**:
  - **搜索策略**: 包含搜索关键词建议、目标网站列表
  - **业务流程场景**: 信贷业务流程中的具体应用场景明细
  - **目标银行和厂商列表**: 搜索覆盖的银行和科技公司范围
  - **推测项目信息**: 基于行业信息的项目预测
  - **搜索结果汇总**: 各智能体搜索任务的详细结果

- **关键信息**:
  - 搜索时间范围: 2024年10月-2025年12月
  - 排除条件: 排除宇信科技相关项目（已覆盖）
  - 重点场景: 智能尽调、信审、财报分析、授信审批、流水分析等

### 3. `final.md` - 最终分析报告
- **文件格式**: Markdown
- **内容长度**: 3830字符
- **报告结构**:
  - **银行分类分析**: 按国有银行、股份制银行、城商行、农商行分类统计
  - **项目汇总**: 总计超过50个大模型应用项目
  - **技术路线分析**: 开源模型主导、云服务合作、混合部署等模式
  - **投资规模分析**: 从百万到千万级别的项目预算分布
  - **时间分布特征**: 2025年为规模化应用高峰期
  - **建设模式**: 自主研发、外部采购、框架合作、共研模式等

## 数据用途与价值

### 业务价值
1. **市场趋势分析**: 了解银行信贷业务数字化转型的最新趋势
2. **竞争格局分析**: 分析各大银行在大模型应用方面的投入和进展
3. **技术路线研究**: 研究不同银行选择的技术路线和合作伙伴
4. **商机识别**: 识别潜在的市场机会和合作可能性

### 技术价值
1. **数据标准化**: 统一的银行大模型项目数据结构
2. **场景分类**: 清晰的信贷业务流程场景分类体系
3. **时间序列分析**: 支持按时间维度的趋势分析
4. **厂商分析**: 科技公司在银行大模型市场的参与情况

## 数据关系
- `search_items.csv` 是原始数据收集结果
- `search_items.json` 记录了搜索策略和过程
- `final.md` 是基于前两者生成的综合分析报告
- 三个文件共同构成完整的银行大模型应用搜索分析体系

## 使用建议
1. **数据更新**: 建议定期执行搜索任务更新数据
2. **质量验证**: 对CSV中的项目信息进行交叉验证
3. **趋势分析**: 结合时间维度分析大模型应用的发展趋势
4. **场景深化**: 针对特定场景进行更深入的数据收集和分析

## 注意事项
1. 数据来源为公开渠道，可能存在信息滞后或不完整的情况
2. 部分项目信息可能涉及商业机密，使用时需注意合规性
3. 建议结合其他数据源进行交叉验证和补充分析
