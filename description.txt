# reasoning_leverage 项目根目录描述文档

## 项目概述
**路径**: `/prog/pweb/reasoning_leverage`
**项目类型**: 推理增强与智能体协作系统
**核心功能**: 提供数据描述生成、代码生成、数学问题求解、答案验证等智能体协作功能
**架构特点**: 基于LangChain智能体框架，支持多级智能体协作、异步处理和状态持久化

## 目录结构总览

### 受Git管理的主要目录
1. **`data_descriptor/`** - 数据描述与处理智能体工具包（7个核心Python模块）
2. **`tools/`** - 通用工具模块（配置管理和对话处理）
3. **`utils/`** - 核心工具和实用模块（智能体管理、工具调用优化、检查点持久化）
4. **`agent_tools/`** - 智能体工具系统（本地工具实现和工具调用管理）
5. **`vendor/`** - 第三方依赖定制化目录（FastMCP框架和供应商补丁）
6. **`logs/`** - 日志和结果数据目录（包含多个数据集的分析结果）
7. **`agentic_codes/`** - 生成的代码目录（当前为空）
8. **`.git/`** - Git版本控制仓库

### 被Git忽略的目录
1. **`.vscode/`** - VS Code IDE配置目录
2. **`agent_checkpoints/`** - 智能体检查点目录
3. **`__pycache__/`** - Python字节码缓存目录（各子目录中均有）

## 子目录详细描述

### 1. data_descriptor/ - 数据描述与处理智能体工具包
**核心功能**: 提供数据描述生成、代码生成、问题求解、答案验证等核心功能
**文件数量**: 7个Python文件
**关键模块**:
- `descript_agent.py` - 数据描述智能体，生成机器可读的描述文档
- `resolver_agent.py` - 数学问题求解智能体系统，支持多智能体协作
- `coding_data_agent.py` - 代码生成智能体，基于数据描述生成Python代码
- `resolver_checker_agent.py` - 解析器检查器智能体，验证答案正确性
- `tool_model.py` - 模型管理模块，提供统一的LLM接口
- `resolver_directly.py` - 直接调用AI模型解答谜题的批量处理脚本
- `web_search.py` - 网络搜索工具模块，集成腾讯Web搜索API

### 2. tools/ - 通用工具模块
**核心功能**: 运行时环境配置管理和对话消息处理
**关键文件**:
- `general_tools.py` - 提供配置读取/写入和对话信息提取功能
**业务角色**: 基础设施层，为上层业务逻辑提供可靠的工具支持

### 3. utils/ - 核心工具和实用模块
**核心功能**: 智能体协作系统的基础设施支持
**关键文件**:
- `talent_training_market.py` - 分层智能体管理系统，支持多级智能体协作
- `tool_call_injector.py` - 工具调用注入中间件，实现工具与上下文解耦
- `model_call_injector.py` - 模型调用中间件，优化工具调用历史管理
- `file_check_point_saver.py` - 文件检查点保存器，实现状态持久化

### 4. agent_tools/ - 智能体工具系统
**核心功能**: 为AI智能体提供本地工具支持和工具调用管理
**子目录结构**:
- `local_tools/` - 本地工具实现
  - `git.py` - Git忽略检查工具
  - `caculator.py` - 安全数学计算工具集（三层安全机制）
- `utils/` - 工具函数和辅助模块
  - `extra_kwargs.py` - 额外参数管理工具，处理元数据注入

### 5. vendor/ - 第三方依赖定制化目录
**核心功能**: 提供第三方库的定制化版本和导入重定向机制
**关键文件**:
- `vendor_patch.py` - 供应商补丁文件，实现模块导入重定向和装饰器增强
- `fastmcp/` - FastMCP框架定制化目录
  - `tools/tool.py` - 核心工具定义文件（包含供应商修改）
  - `tools/tool copy.py` - 工具副本文件（包含调试痕迹）

### 6. logs/ - 日志和结果数据目录
**核心功能**: 存储智能体协作的日志文件和分析结果
**目录结构**:
- `resolve/` - 谜题求解结果目录
  - `analysis_resolve_50.json` - 50个问题的智能体协作分析数据
  - `deepseek/check_result_batch.json` - 50个谜题的答案验证结果
  - `puzzle_*/` - 各个谜题的详细求解目录
- `aime_20260221/` - AIME 2026竞赛问题求解目录
  - `analysis_aime_2026_30.json` - 30个问题的分析数据
  - `puzzle_*/` - 各个问题的求解目录
- `search_res/` - 搜索结果数据目录
  - `search_items.csv` - 银行信贷业务流程中大模型应用场景搜索结果（131行）
  - `search_items.json` - 搜索策略与过程记录
  - `final.md` - 最终分析报告
- `sepc_call_llm_20260403.log` - 模型调用日志文件

### 7. agentic_codes/ - 生成的代码目录
**当前状态**: 空目录
**预期用途**: 存储由`coding_data_agent.py`生成的Python代码文件

## 数据文件分析

### 1. 智能体协作分析数据
**文件位置**: `logs/resolve/analysis_resolve_50.json`
**数据结构**: JSON数组，包含50个问题的分析结果
**核心字段**:
- `question_id`: 问题编号（1-50）
- `total_agents`: 使用的智能体总数
- `max_depth`: 智能体树的最大深度
- `hire_cost`: 雇佣成本（token数）
- `execution_cost`: 执行成本（token数）
- `leverage`: 杠杆率（执行成本/雇佣成本）
- `mails_length`: 邮件通信长度
- `shared_length`: 共享黑板内容长度

**统计信息**: 
- 平均智能体数: 7.62
- 平均雇佣成本: 28628.18 tokens
- 平均执行成本: 379842.18 tokens
- 平均杠杆率: 13.27

### 2. 答案验证结果数据
**文件位置**: `logs/resolve/deepseek/check_result_batch.json`
**数据结构**: JSON对象，包含50个谜题的验证结果
**核心字段**:
- `puzzle_{n}`: 每个谜题的验证结果
  - `check_result`: 布尔值，表示答案是否正确
  - `description`: 详细的验证描述和错误分析

**统计信息**: 50个谜题中，约60%的答案正确，40%存在错误

### 3. 银行大模型应用搜索数据
**文件位置**: `logs/search_res/search_items.csv`
**数据量**: 131行记录
**字段结构**:
- `BankName`: 银行名称
- `Scene`: 应用场景（智能尽调、信审、财报分析等）
- `Description`: 项目描述
- `Type`: 采购类型（招标采购、公开招标等）
- `TimeCycle`: 时间周期（2024-2025年）
- `Vendor`: 供应商（科技公司名称）

**数据特点**: 覆盖国有银行、股份制银行、城商行、农商行等各类银行机构

## 业务工作流程

### 1. 数据描述流程
```
用户需求 → descript_agent → 目录扫描 → 文件采样 → 生成description.txt
```

### 2. 问题求解流程
```
数学问题 → resolver_agent → 多智能体协作 → 结果验证 → 成本分析 → 结果保存
```

### 3. 代码生成流程
```
用户需求 → coding_data_agent → 数据探索 → 代码生成 → 状态机管理 → 代码保存
```

### 4. 答案验证流程
```
用户答案 → resolver_checker_agent → 标准答案对比 → 等价性验证 → 结果保存
```
