# 游戏胜负状态分析

## 1. 问题描述
初始黑板上有一个正整数n（1 ≤ n ≤ 10000）。玩家轮流操作：擦除一个大于1的整数k，用5个正整数替换，这5个数的和等于k且两两差不超过1。无法操作者输。双方最优策略，玩家1先手获胜。

## 2. 关键发现

### 2.1 操作可行性
k可以进行操作当且仅当k ≡ 0, 1, 4 (mod 5)。具体分解：
- k ≡ 0 (mod 5): 分解为5个k/5
- k ≡ 1 (mod 5): 分解为4个(k-1)/5和1个(k-1)/5+1
- k ≡ 4 (mod 5): 分解为4个(k+1)/5和1个(k+1)/5-1

### 2.2 游戏可加性
由于每个数字的操作独立，整个游戏的Grundy数等于各个数字Grundy数的异或和。

### 2.3 单个数字的Grundy数
定义g(k)为数字k的Grundy数：
- g(k)=0 如果k不可操作（k≤1或k≡2,3 mod 5）
- 否则，g(k)=mex{g(k₁)⊕g(k₂)⊕g(k₃)⊕g(k₄)⊕g(k₅)}，其中(k₁,...,k₅)是k的合法分解

### 2.4 递归关系推导
分析分解的异或和：
1. k≡0 mod 5: 5个g(k/5)的异或 = g(k/5)（5是奇数）
2. k≡1 mod 5: 4个g((k-1)/5)和1个g((k-1)/5+1)的异或 = g((k-1)/5+1)（4个相同数异或为0）
3. k≡4 mod 5: 4个g((k+1)/5)和1个g((k+1)/5-1)的异或 = g((k+1)/5-1)

因此，g(k)=1当且仅当上述异或和为0，即：
- k≡0 mod 5: g(k)=1 ⇔ g(k/5)=0
- k≡1 mod 5: g(k)=1 ⇔ g((k-1)/5+1)=0
- k≡4 mod 5: g(k)=1 ⇔ g((k+1)/5-1)=0（需(k+1)/5≥2）

### 2.5 边界条件
- g(1)=0
- g(4)=0（虽然4≡4 mod 5，但(4+1)/5=1，1-1=0无效）

## 3. 算法实现
基于递归关系计算g(k) for k=1..10000：

```python
def compute_g(max_n):
    g = [0] * (max_n + 1)
    for k in range(1, max_n + 1):
        if k == 1:
            g[k] = 0
            continue
        mod5 = k % 5
        if mod5 == 2 or mod5 == 3:
            g[k] = 0
            continue
        if mod5 == 0:
            x = k // 5
            g[k] = 1 if g[x] == 0 else 0
        elif mod5 == 1:
            x = (k - 1) // 5
            g[k] = 1 if g[x+1] == 0 else 0
        elif mod5 == 4:
            x = (k + 1) // 5
            if x >= 2:
                g[k] = 1 if g[x-1] == 0 else 0
            else:
                g[k] = 0
    return g
```
