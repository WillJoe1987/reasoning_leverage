# data_descriptor 目录描述文档

## 目录概述

**路径**: `/prog/pweb/reasoning_leverage/data_descriptor`
**功能定位**: 数据描述与处理智能体工具包，提供数据描述生成、代码生成、问题求解、答案验证等核心功能
**文件数量**: 7个Python文件
**忽略目录**: `__pycache__`（Python缓存目录，自动忽略）

## 文件详细描述

### 1. descript_agent.py
**文件大小**: 10,005 字符
**核心功能**: 数据描述智能体，为指定目录生成机器可读的描述文档（description.txt）
**主要组件**:
- `descriptor_agent` 类：继承自 `SubAgent`，负责单个目录的描述任务
- `walk_run()` 函数：异步遍历目录树并执行描述任务
- 工具函数：`signature_python_module()`, `get_python_includes()`, `get_python_object_code()`

**输入/输出接口**:
- 输入：目录路径字符串
- 输出：生成 `description.txt` 描述文件
- 系统提示：包含详细的任务目标、描述内容要素、工具使用指引

**业务角色**: 数据描述系统的核心执行引擎，负责创建分层信息地图

### 2. resolver_agent.py
**文件大小**: 10,005 字符
**核心功能**: 数学问题求解智能体系统，解决和证明复杂的数学问题
**主要组件**:
- `resolver_agent` 类：继承自 `SubAgent`，支持多智能体协作
- 验证机制：通过独立的验证智能体确保结果的正确性、最优性和完整性
- 成本分析：跟踪和分析智能体协作的成本效益

**输入/输出接口**:
- 输入：数学问题文本、数据集名称
- 输出：结果文件、智能体树结构JSON文件、共享黑板内容、邮件通信记录
- 分析输出：成本分析报告、智能体层级统计、通信量统计

**业务角色**: 数学问题求解的核心模块，通过智能体协作实现复杂数学问题的自动化求解

### 3. resolver_directly.py
**文件大小**: 4,589 字符
**核心功能**: 直接调用AI模型解答谜题的批量处理脚本
**主要组件**:
- `run_data_sample()` 函数：异步处理指定范围内的数据样本
- `process_one_sample()` 函数：处理单个数据样本的内部异步函数
- 并发处理：采用分批并发的方式提高处理效率

**输入/输出接口**:
- 输入：开始索引、结束索引
- 输出：AI模型的回答结果文件，保存到 `logs/resolve_direct/` 目录
- 模型支持：支持多种AI模型（deepseek、qwen、baidu、gpt、gemini）

**业务角色**: 连接AI模型和谜题数据集的关键桥梁，实现批量数据处理

### 4. resolver_checker_agent.py
**文件大小**: 4,580 字符
**核心功能**: 解析器检查器智能体，验证用户答案相对于标准答案的正确性
**主要组件**:
- `resolver_checker_agent` 类：继承自 `SubAgent`，负责单个问题的答案检查
- `save_result` 工具函数：将检查结果保存到JSON文件
- `load_user_answer()` 函数：从指定路径加载用户答案文件
- `run_checker_agent()` 函数：异步批量运行检查器

**输入/输出接口**:
- 输入：问题索引、数据集名称、检查任务描述
- 输出：检查结果保存到 `logs/{dataset}/{model_type}/check_result_batch.json`
- 验证逻辑：检查答案正确性，对于数学表达式验证其等价性

**业务角色**: 答案验证和质量控制模块，确保解析器生成的答案质量

### 5. tool_model.py
**文件大小**: 未指定（中等规模）
**核心功能**: 模型管理模块，提供统一的大语言模型（LLM）接口
**主要组件**:
- `DeepSeekChatOpenAI` 类：自定义的DeepSeek模型包装器，修复tool_calls格式问题
- `get_model_by_name()` 函数：根据模型名称返回对应的模型实例
- `get_default_model()` 函数：获取默认的DeepSeek模型
- 模型支持：deepseek、qwen、baidu、gpt、gemini

**输入/输出接口**:
- 输入：模型名称字符串、环境变量配置API密钥
- 输出：配置好的LangChain Chat模型实例
- 日志输出：在 `{project_root}/logs/sepc_call_llm_{日期}.log` 中记录交互日志

**业务角色**: 基础服务提供者，为上层代理提供统一的模型调用接口

### 6. coding_data_agent.py
**文件大小**: 10,005 字符
**核心功能**: 代码生成智能体模块，基于数据描述文件自动生成Python代码
**主要组件**:
- `data_spaces` 类：数据空间管理器，定义和管理不同的数据存储区域
- `coding_data_agent` 类：代码生成智能体，基于用户需求和数据描述生成代码
- 状态机管理：管理复杂的代码提取状态（状态码0-3）
- 分段代码处理：支持长代码的分段生成和续写

**输入/输出接口**:
- 输入：用户需求描述、数据空间对象、模块名称
- 输出：生成的Python代码文件，保存到 `agentic_codes/` 目录
- 系统提示：包含严谨的代码生成原则和工作流

**业务角色**: 代码生成引擎，将结构化的数据描述转化为可执行的Python代码

### 7. web_search.py
**文件大小**: 未指定（中等规模）
**核心功能**: 网络搜索工具模块，集成腾讯Web搜索API
**主要组件**:
- `WebSearchTool` 类（推断）：基于腾讯API的Web搜索工具
- HTML解析：使用BeautifulSoup解析搜索结果
- 数据持久化：使用SQLite数据库存储搜索历史
- 异步处理：支持异步操作提高效率

**输入/输出接口**:
- 输入：搜索查询字符串、结果数量、时间范围等参数
- 输出：结构化的搜索结果JSON格式
- 依赖：腾讯API密钥、网络连接

**业务角色**: 网络数据收集模块，为其他Agent提供外部数据支持

## 模块间依赖关系

### 内部依赖链
```
tool_model.py（基础服务）
    ↓
descript_agent.py（核心引擎）
    ├── resolver_agent.py（问题求解）
    ├── resolver_directly.py（直接处理）
    ├── resolver_checker_agent.py（答案验证）
    ├── coding_data_agent.py（代码生成）
    └── web_search.py（网络搜索）
```

### 外部依赖
1. **LangChain生态**: 智能体框架、工具系统、消息格式
2. **HuggingFace数据集**: 用于加载AMO-Bench等数据集
3. **腾讯Web搜索API**: 用于网络搜索功能
4. **多种AI模型API**: DeepSeek、Qwen、百度文心、GPT、Gemini

## 业务工作流程

### 1. 数据描述流程
```
用户需求 → descript_agent → 目录扫描 → 文件采样 → 生成description.txt
```

### 2. 问题求解流程
```
数学问题 → resolver_agent → 多智能体协作 → 结果验证 → 成本分析 → 结果保存
```

### 3. 代码生成流程
```
用户需求 → coding_data_agent → 数据探索 → 代码生成 → 状态机管理 → 代码保存
```

### 4. 答案验证流程
```
用户答案 → resolver_checker_agent → 标准答案对比 → 等价性验证 → 结果保存
```

## 使用示例

### 1. 生成目录描述
```python
from data_descriptor.descript_agent import walk_run
import asyncio

async def describe_all():
    await walk_run("/base/path")
```

### 2. 求解数学问题
```python
from data_descriptor.resolver_agent import resolver_agent

agent = resolver_agent(
    agent_num="puzzle_1",
    question="数学问题文本",
    dataset="resolve"
)
response = await agent.execute()
```

### 3. 生成代码
```python
from data_descriptor.coding_data_agent import coding
import asyncio

asyncio.run(coding(
    input_requirement="编写股票因子计算代码",
    data_space="market",
    module_name="stock_factor_calculator"
))
```

## 注意事项

2. **Git管理**: 优先检查git忽略规则，仅描述受git管理的内容
3. **Python文件分析**: 对于每个 `.py` 文件，可以雇佣子智能体进行详细分析
4. **效率优化**: 避免重复采样，对于规律命名的文件选取典型代表分析
5. **分层描述**: 上层目录概括，下层目录具体，创建分层信息地图
