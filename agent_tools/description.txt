## 子目录详情

### 1. local_tools/ - 本地工具实现
**路径**: `local_tools/`
**状态**: 受git管理，已有description.txt文件
**内容**: 包含2个Python工具文件，为智能体系统提供本地工具支持

#### 核心文件分析

##### git.py - Git忽略检查工具
**功能**: 判断指定文件或目录是否被git忽略
**工具类型**: LangChain工具装饰器（@tool）
**检查方法**: 双重检查策略
1. 优先使用`git check-ignore -v`命令
2. 回退到手动解析.gitignore文件

**接口**:
- 函数: `is_git_ignored(path: str) -> dict`
- 参数: `path` - 要检查的文件或目录路径
- 返回值: 包含详细信息的字典结构

**业务角色**:
- 智能体工具: 为AI智能体提供检查文件是否被git忽略的能力
- 代码管理: 帮助智能体理解项目的代码组织结构
- 文件过滤: 用于过滤掉不应该处理的git忽略文件

##### caculator.py - 安全数学计算工具集
**功能**: 安全数学计算工具集，包含三个核心功能：
1. **安全数学表达式计算** - 支持单行数学表达式的安全计算
2. **迭代任务求解** - 支持在指定范围内迭代计算表达式
3. **纯数学代码沙盒** - 提供安全的Python代码执行环境

**类结构**:
- `SafeMathEvaluator`类 - 安全数学表达式计算器
- `IterativeTaskSolver`类 - 迭代任务求解器
- `PureMathSandbox`类 - 纯数学代码沙盒

**安全特性**:
- 白名单函数机制：只允许预定义的数学函数
- AST语法树检查：递归检查所有节点
- 禁止导入、文件、网络、系统操作
- 限制执行时间、循环次数和递归深度

**工具接口**:
- `get_calculator_tool()` - 返回计算器工具集
- `get_sandbox_tool()` - 返回数学沙盒工具

### 2. utils/ - 工具函数和辅助模块
**路径**: `utils/`
**状态**: 受git管理，已有description.txt文件
**内容**: 包含agent_tools系统的工具函数和辅助模块，主要用于处理工具调用中的参数注入和元数据管理

#### 核心文件分析

##### extra_kwargs.py - 额外参数管理工具
**功能**: 提供工具调用中额外参数（extra kwargs）的提取和管理功能，主要用于从工具调用请求中提取需要注入的元数据要求

**核心函数**:
1. `get_key_injects(kwargs: dict, key: str)`
   - 功能: 检查请求中的工具调用，获取指定键需要注入的元数据要求列表
   - 返回值格式: `[{ "key": key, "required": bool }, ...]`
   - 若无注入要求，返回 `None`

2. `get_all_injects(kwargs: dict)`
   - 功能: 检查请求中的工具调用，获取所有需要注入的元数据要求
   - 返回值格式: `{ key: value, ... }`
   - 从 `kwargs` 字典的 `"kwargs"` 键中提取所有注入要求

**代码实现**:
```python
def get_key_injects(kwargs:dict, key:str):
    """
        检查请求中的工具调用，获取需要注入的元数据要求列表。
        返回格式：[{ "key": key, "required": bool }, ...]
        若无注入要求，返回空列表。
    """
    return get_all_injects(kwargs).get(key, None)

def get_all_injects(kwargs:dict):
    """
        检查请求中的工具调用，获取所有需要注入的元数据要求。
        返回格式：{ key: value, ... }
    """
    args = kwargs.get("kwargs", {})
    return args
```

**在业务流中的角色**:
1. **元数据注入管理**: 在agent_tools系统中，当智能体调用工具时，可能需要注入额外的元数据（如用户上下文、会话信息等）
2. **参数提取**: 从工具调用请求中提取预定义的注入要求
3. **条件检查**: 根据注入要求决定是否需要在工具调用前注入特定元数据

## 使用建议

### 1. 工具调用流程
1. **工具选择**: 根据需求选择适当的工具（git检查、数学计算等）
2. **参数准备**: 使用 `utils/extra_kwargs.py` 检查是否需要注入额外参数
3. **安全执行**: 对于数学计算，使用 `caculator.py` 的安全机制
4. **结果处理**: 处理工具返回的结果

### 2. 安全注意事项
1. **git.py**: 确保正确处理git忽略规则，避免处理不应该处理的文件
2. **caculator.py**: 虽然提供了多层安全机制，但仍需谨慎处理用户输入
3. **extra_kwargs.py**: 确保参数注入的正确性和安全性

### 3. 代码复用建议
1. **git检查**: 当智能体需要检查文件是否应该被处理时，复用 `git.py`
2. **数学计算**: 当需要进行数学计算、算法验证或数值模拟时，复用 `caculator.py`
3. **参数管理**: 当需要管理工具调用的额外参数时，复用 `extra_kwargs.py`
